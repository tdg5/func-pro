Immutability

Immutabe Data Structures

  • Immutable data types, when updated, do not update in-place, but instead yield a new object with the updated data.
  • Can actually be very efficient
    • New objects can point to existing data cheaply
    • Compilers can make optimiaztions that wouldn't be safe with mutable data
  • Often used with and similar in benefit to pure functions:
    • Easier to understand, predict and reason about.
    • Facilitates code reuse.
    • Facilitates testing.
    • Variations in timing and order of calls does not affect result.
  • Immutable.js brings immutable collections to JavaScript.
