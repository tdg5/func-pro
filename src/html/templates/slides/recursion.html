Recursion

  • Recursive functions invoke themselves repeatedly until some base case is reached at which point a value is returned.
  • Recursion requires maintaining a stack which can be problematic in languages that don't support tail-call optimization .
    • Some aspects of very deep recursion can be faked through the use of higher-order functions. For example, the y-combinator can be represented both functionally and imperatively.
  • Looping in functional languages is usually accomplished via recursion.
